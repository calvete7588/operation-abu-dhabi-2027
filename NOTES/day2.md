## Day 2 – GitHub & Remote Repository Setup
**Date:** July 7, 2026

## Objective
Connect my local development environment to GitHub to establish my public professional portfolio.

## Key Concepts Learned
* **Git vs. GitHub:** Git is the local version control software on my PC. GitHub is the cloud platform that hosts my repositories and displays my work to recruiters.
* **Syntax Rule:** Every Git command must strictly begin with the word `git`.

## Technical Vocabulary
* **Repository:** A digital storage folder for a project's files and history.
* **Commit:** A saved snapshot of the project at a specific moment.
* **Push:** Sending local changes up to the remote server on GitHub.

## Commands Executed & Explained
* `git init` ➔ Initializes a new local Git tracking database in the project folder.
* `git add .` ➔ Stages all modified files, preparing them to be snapshotted.
* `git commit -m "message"` ➔ Records the staged snapshot with a descriptive note.
* `git remote add origin <URL>` ➔ Links the local repository to the specific cloud repository on GitHub.
* `git push -u origin main` ➔ Uploads the local commits to the primary cloud branch for the first time.
